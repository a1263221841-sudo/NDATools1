# [OPEN] WebSocket Handshake Drop

## 会话信息
- session_id: `websocket-handshake-drop`
- 时间: 2026-07-20
- 症状: 连接 `ws://echo.websocket.events` 时发送握手请求后立即断开，没有进入“握手成功”状态。

## 当前观察
- 已看到日志:
  - `正在连接 ws://echo.websocket.events`
  - `发送握手请求`
  - `WebSocket 已断开`

## 初始假设
1. 握手请求中的 `Host` / `path` / `port` 头格式不符合服务端要求，服务端直接关闭连接。
2. 目标公网回显服务当前不可用、已变更、或不再接受当前域名上的明文 `ws` 连接。
3. 套接字在收到服务端响应前就进入断开流程，但当前代码没有把断开前的错误信息和原始响应头记录出来。
4. 握手响应其实已返回，但 `readyRead` 处理和 `disconnected` 顺序导致我们没来得及记录响应内容。
5. 当前实现缺少 `errorOccurred` 级别的运行时证据，导致无法区分“DNS/网络失败”和“服务端主动断开”。

## 计划
1. 仅添加最小化调试日志，不改业务逻辑。
2. 复现问题，收集连接、错误、断开、原始响应头证据。
3. 基于证据判断根因后，再做最小修复。
